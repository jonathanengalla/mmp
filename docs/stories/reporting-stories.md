# Reporting User Stories (Thin-Sliced)

1. **Story:** Admin views member roster report.  
   **Acceptance Criteria:**  
   - Columns: name, email, status, role.  
   - Filters by status/role.  
   - Export to CSV available.

2. **Story:** Admin views dues collection summary.  
   **Acceptance Criteria:**  
   - Shows total billed, collected, outstanding.  
   - Date range filter.  
   - Data updates on filter apply.

3. **Story:** Admin views donation summary.  
   **Acceptance Criteria:**  
   - Totals by campaign.  
   - Filters by date/campaign.  
   - Export respects filters.

4. **Story:** Admin views event attendance report.  
   **Acceptance Criteria:**  
   - Select event to view attendance counts.  
   - Shows registered vs checked-in.  
   - Export option available.

5. **Story:** Admin views payment transactions report.  
   **Acceptance Criteria:**  
   - Columns: amount, status, gateway, member, date.  
   - Filters by status/date range.  
   - Paginated results.

6. **Story:** Admin runs member growth trend.  
   **Acceptance Criteria:**  
   - Chart shows new vs churned over time.  
   - Time granularity selectable (month/quarter).  
   - Hover reveals counts.

7. **Story:** Admin exports selected report to PDF.  
   **Acceptance Criteria:**  
   - PDF uses tenant branding.  
   - Matches on-screen filters.  
   - Download completes without error.

8. **Story:** Admin schedules recurring report email.  
   **Acceptance Criteria:**  
   - Choose report, frequency, recipients.  
   - Email includes link or attachment.  
   - Schedule status visible and editable.

9. **Story:** Admin saves report filter as a view.  
   **Acceptance Criteria:**  
   - Saved view stores filters/sorting.  
   - Accessible in saved views list.  
   - Can rename or delete view.
